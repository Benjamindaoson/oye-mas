## 一、产品一句话定位

> **「有了」是你的专属 AI 工作团队。**

第一期重点针对**内容引流**和**营销**场景。整个产品的外观和交互**80% 像微信**。

### 用户为什么用(一句话 + 4 点)

**一句话**:用更低的成本、更短的时间,拿到一次到位的专业结果。

具体:

| 价值 | 说明 |
|------|------|
| 操作便捷 | 自然语言下达需求,不学 Prompt 工程 |
| 响应迅速 | 多 Agent 并行协作,不卡顿 |
| 一次到位 | 成果直接可用,不改或仅小改 |
| 降本增效 | 替代专业团队(设计师 / 文案 / 剪辑) |

### 与现有 AI(GPT/Claude/DeepSeek)的差异

**现有 AI 的问题**:

1. 多次交互才能拿到结果,且常不满意 → 反复修改的时间 ≈ 自己做
2. 直接调大模型成本高,中小用户不友好

**「有了」的解决方式**:

主编排 Agent 一次性理解意图 → 自动调 Skill 工作流 → 多 Agent 协同交付完整成品。**用户从"和 AI 反复来回"变成"对 AI 团队下达任务"。**

> ⚠️ 文案选择:同时含**技术差异**("多 Agent vs 单一 LLM")与**产品差异**("DIY vs 团队交付")。两层都保留;对外宣传重点用**产品差异**(更好懂)。

## 二、核心交互

### 2.1 三栏布局

| 栏 | 内容 |
|----|------|
| 左栏 | 会话列表 + 应用导航 |
| 中栏 | 对话主战场,群聊框 |
| 右栏 | **执行流**(核心差异化)|

### 2.2 右栏执行流

- 让用户实时看到 AI 团队怎么协作
- **三层抽屉式进度**:阶段总览 / 子任务结构 / 实时细节
- 状态符号:**✓ 完成** / **⟳ 进行中** / **○ 未开始**

### 2.3 Agent 的用户呈现

| 技术编号 | 用户面角色名 | 备注 |
|---------|-----------|------|
| 主编排 | **总裁助理** | 唯一中央调度 |
| Agent 1 | 研究员 / 文案师 | 同一 Agent 两个角色,按场景切换 |
| Agent 2 | 设计师 | |
| Agent 3 | 影音师 | |
| Agent 4 | 文档专员 | |

> **关键**:群内每个 Agent 是一个数字员工。**不同群岗位名字不同**,但底层技术 Agent 通用。

### 2.4 群的形态

- **通用群**:装备全部 4 个 Agent,处理日常需求
- **专门群**:装备特定 Skill 包(反诈视频群、电商作图群)
- 每个群独立总裁助理在场
- 群头像:**默认"我的世界"像素风 / NFT 风格**

### 2.5 @ 调用机制

| 类型 | 行为 |
|------|------|
| `@人(AI 员工)` | 列出当前群所有 Agent,选中插入 `@研究员` |
| `@文件(素材库)` | 从素材库选文件,插入 `@文件名.jpg` |
| `@提示词(知识库)` | 从收藏 Prompt 选,内容自动展开到输入框 |

### 2.6 视觉风格

- 三栏极简布局,大量留白
- **微信绿 #07c160 唯一强调色**
- 线条 SVG 图标,**无 emoji**
- 头像统一"我的世界"像素风 / NFT 风格

## 四、群内 Agent 交互细节

### 4.1 设计哲学

> 每个 Agent 不仅是工具,更是用户可了解、信任、深入互动的**数字员工**。
> 通过履历卡片、悬停交互、私聊机制实现 AI 角色的人格化。

### 4.2 悬停交互 — Agent 履历卡片

**触发**:鼠标悬停 Agent 头像 0.3 秒后浮现

**卡片内容**:

| 区域 | 内容 |
|------|------|
| 基础信息 | 头像 + 名称 + 一句话能力定位 + 背后模型 |
| 专业标识 | 3-5 个领域标签 + 进修记录(如有)|
| 服务统计 | 个人("已为你完成 47 次")+ 平台("累计 120 万次")|
| 操作按钮 | [私聊] [查看更多] |

### 4.3 私聊交互

**触发(两种等效)**:

- 双击 Agent 头像
- 点击履历卡片 [私聊] 按钮

**私聊特征**:

- 不启动完整 Skill 工作流
- 不调度其他 Agent
- 响应更快(跳过编排环节)
- 对话历史独立(与群聊互不污染)
- 仍可调用单 Agent 的轻量 Skill

**适用场景**:

- 让"文案师"快速改一句话
- 让"研究员"查一个数据
- 让"设计师"调整一张图的细节
- 让"文档专员"格式化一份表格

### 4.4 群聊 vs 私聊边界

| 维度 | 群聊 | 私聊 |
|------|------|------|
| 任务复杂度 | 多步骤、跨能力 | 单一能力、快速响应 |
| Agent 数量 | 多 Agent 协同 | 单 Agent 直接执行 |
| 走 Skill 工作流 | 是 | 否(轻量 Skill 可用)|

### 4.5 Agent 名片分享(延伸)

用户可将 Agent 履历卡片作为"名片"分享:

- 朋友点击 → 看到履历 → 一键进入私聊 → 引导注册
- **产品意义**:零成本获客飞轮,AI 员工社交化

## 五、卡片式分享 — 产品的获客飞轮

### 5.1 设计原则

产品核心是**群聊**,不是单个 Agent。

**分享优先级**:**群 > 成果 > 单 Agent**

### 5.2 三种分享形态

#### ① 群名片分享(核心)

| 区域 | 内容 |
|------|------|
| 基础信息 | 群头像(4 色块) + 群名称 + 群简介 |
| 团队组成 | Agent 列表 + 每个 Agent 的角色 |
| 装备 Skill | 该群订阅或预置的 Skill 列表 |
| 战绩数据 | 累计交付成果数 + 群成就 |
| 操作按钮 | [加入此团队] [查看成果] [关注创作者] |

**价值**:朋友一键复制整个团队配置(Agent + Skill + 偏好),零摸索成本。

#### ② 成果分享(核心)

| 区域 | 内容 |
|------|------|
| 成果本体 | 缩略图 / 预览 / 可播放 |
| 产出溯源 | 来自哪个群、哪些 Agent、用什么 Skill、耗时多久 |
| 操作按钮 | [下载] [做同款] [查看创作者] |

**[做同款] 行为**:

- 自动创建相同配置的群
- 引导用户填入自己的参数
- 5 分钟产出用户自己的版本

**价值**:从"看到成果"到"拥有同款"路径极短,**成果即广告**。

#### ③ 单 Agent 名片(次要)

详见 §4.5,非主推形态。

### 5.3 三种分享对比

| 形态 | 路径 | 扩散级别 |
|------|------|---------|
| 群名片 | 朋友复制团队 | **团队级扩散** |
| 成果 | 朋友做同款 | 成果驱动转化 |
| Agent 名片 | 朋友私聊 Agent | 工具级体验 |

### 5.4 增长飞轮

```
用户产出好成果 → 分享群/成果 → 朋友看到
   ↑                              ↓
朋友再次分享     ←     朋友也产出   ←  一键做同款 / 加入团队
```

**核心**:每次分享 = 产品价值传递 + 产品功能演示。

## 六、系统核心竞争力

### 6.1 一句话

意图理解、意图澄清、任务编排——把用户**模糊的需求**精准转化为**结构化任务**,并交付**符合甚至超出预期**的结果。

### 6.2 系统两部分

| 部分 | 内容 |
|------|------|
| **不变的部分** | 多智能体框架(主编排 Agent + 4 个分任务 Agent)|
| **变的部分** | Prompt(单步骤,小颗粒,用户可收藏)+ Skill(完整工作流,大颗粒,多步骤多 Agent)|

### 6.3 Skill 库设计

- **来源**:平台预置 + 创作者贡献
- **用户不直接配置 Skill**
- 主编排 Agent 理解意图后,**自动从 Skill 库匹配并调用**,分配给对应 Agent 执行

### 6.4 创作者飞轮

```
用户对总裁助理说:"我要成为创作者"
  ↓
总裁助理引导:
  ① 自动分析用户最近的工作流
  ② 自动总结成 Skill 草稿
  ③ 用户审核确认
  ④ 平台审核(质量 + 不含违规 prompt)
  ⑤ 发布到 Skill 市场
```

**审核机制不能省**:避免市场被低质 / 违规 Skill 污染。

## 七、三种工作模式(用户选择)⭐ 新增

总裁助理在用户开始工作前**确认以哪一种模式开始**:

| 模式 | 英文名 | 行为 |
|------|-------|------|
| **讨论模式** | plan mode | 先聊清楚需求 / 看参考 / 不真做 |
| **询问模式** | ask mode | 边做边问,用户全程参与决策 |
| **自动模式** | auto mode | 直接开干,关键节点才打断用户 |

> ⚠️ **与 v2.0 ADR-008 / ADR-010 的关系**:见末尾"差异点 ②"。

## 八、首次进入体验 ⭐ 新增

用户第一次进入「有了」**自动进入一个群**,叫做:

> **「你的专属 AI 工作团队」**

### 进入动画(像加入微信群)

依次显示:

```
✓ 总裁助理 加入群聊
✓ HR 加入群聊
✓ 财务经理 加入群聊
（其他 Agent 按需后续介绍）
```

### 总助、HR、财务经理依次自我介绍

- 总裁助理:"你好,我是你的总裁助理,负责理解你的需求并调度团队..."
- HR:"我是 HR,负责……"(具体角色对应 Agent 1-4 中的某一个)
- 财务经理:"我是财务经理,管账目和数据..."

### 总助介绍三种工作模式 + 模式选择卡

```
总裁助理:在我们开始干活前,你想用哪种模式?

[💭 讨论模式 plan]  先聊聊,梳理需求
[❓ 询问模式 ask]   边做边问,你来决策
[🚀 自动模式 auto]  直接开干,关键时打断你
```

### 群内 Agent 互动要"像真实公司工作群"

- Agent 之间的互动和对话**多一些拟人化**
- 接近真实人类公司工作群的样子(打招呼、互相 cue、交接工作)
- **不是冷冰冰的工具机器**

> ⚠️ **与 v2.0 的"群成员栏"概念不同**:v2.0 是 4 个分任务 Agent;这里是"HR / 财务经理"角色。见末尾"差异点 ①"。

## 九、典型应用场景

### 9.1 场景 1:反诈视频制作

**总裁助理调用「反诈视频制作」Skill**

| 步骤 | Agent | 内容 | 输出 |
|------|-------|------|------|
| 1 案例调研 | Agent 1 → DeepSeek | 调研 10 条 {年份} 涉及 {骗局类型} 的案件,整理涉案金额 / 摘要 / 抖音风标题 / 来源 URL / 配图 URL | xlsx |
| 2 脚本生成 | Agent 1 | 基于 xlsx 生成视频脚本;主编排发起选择题澄清(≤5 轮) | docx |
| 3 图片处理 | Agent 1 + Agent 2 | 从 xlsx 下载图片,自动命名,质量检查 | 5 张配图 |
| 4 配乐生成 | Agent 3 | 基于脚本情绪选 BGM | mp3 |
| 5 自动剪辑 | Agent 3 | 整合脚本 / 图 / 配音 / BGM / 字幕 | mp4 |

**最终交付**:mp4 + xlsx + docx + 5 张配图 + mp3

### 9.2 场景 2:电商详情图制作

**总裁助理调用「电商详情图」Skill**

| 步骤 | Agent | 内容 | 输出 |
|------|-------|------|------|
| 1 素材输入 | 用户 | 上传商品图 + 卖点 + 风格选择 | - |
| 2 风格分析 | Agent 2 | 分析所选风格的视觉特征 | 风格指引 |
| 3 文案撰写 | Agent 1 | 基于风格指引和卖点,写 5 段文案 | 5 段文案 |
| 4 分段图生成 | Agent 2 | 5 张分段图,**第 1 张为视觉锚点**,后续保持风格统一 | 5 张分段图 |
| 5 长图拼接 | Agent 4 | 垂直拼接为 1 张完整长图 | 长图 |
| 6 质量校验 | Agent 2 | 检查文字清晰度、风格一致性、商品突出度,不达标自动重做 | - |

**最终交付**:1 张精美的电商详情长图

### 9.3 关键设计共性(从两个场景提炼)

| 共性 | 说明 |
|------|------|
| 用户输入是模糊的 | "做反诈视频"、"做朴朴风格" |
| 主编排负责理解和编排 | 自动匹配 Skill + 分配任务给对应 Agent |
| 多 Agent 协同 | 反诈:Agent 1+2+3;电商:Agent 1+2+4 |
| 关键决策点向用户澄清 | 永远选择题,≤5 轮,不让填空 |
| 最终交付完整产物 | 不是"半成品+用户组装",是成片或成图,拿到就能用 |

### 9.4 关键技术原则

- **Prompt 是 Skill 内置的**,含变量插槽
- **主编排自动填充变量**并调用
- **用户只说自然语言**,不写 prompt

## 附录 A:与 v2.0 架构文档的差异点(待团队确认)

整理过程中发现以下与现有 v2.0 文档不一致的地方,**需要团队拍板**:

### ⚠️ 差异点 ① 首次进入的 Agent 角色

| 来源 | 首次进入群 | 出场 Agent |
|------|----------|-----------|
| **本整理版**(用户最新输入)| "你的专属 AI 工作团队" | 总裁助理 + **HR + 财务经理** |
| **v2.0 文档** | 总裁助理主会话 | 仅总裁助理 |

**问题**:

- HR / 财务经理是 Agent 1-4 中哪一个的别名?还是新概念?
- 如果是新概念,是否要扩展为 6 个 Agent?
- 如果只是用户面角色名,**对应映射是什么**?

**建议**:HR ↔ Agent 1(研究员/文案,涉及人员档案)、财务经理 ↔ Agent 4(文档专员,涉及表格)。**或者只在"首次欢迎群"用这两个角色作 demo,日常工作群仍用研究员/设计师/影音师/文档专员**。

---

### ⚠️ 差异点 ② 工作模式数量

| 来源 | 模式 |
|------|------|
| **本整理版** | **3 种**:讨论(plan)/ 询问(ask)/ 自动(auto) |
| **v2.0 文档(ADR-008)** | **2 种**:讨论 / 干活 |

**问题**:

- "询问模式 ask"是新加的中间态吗?
- "自动模式 auto"对应原"干活模式"?
- 三模式如何映射到 v2.0 的 `conversation.mode` 字段?

**建议**:三选一:

- **方案 A**:升级为 3 种模式,改 `conversation.mode` 枚举(plan / ask / auto / main_session / private_chat)
- **方案 B**:讨论 = plan,干活 = auto,**询问模式合并到干活模式内的"中断频率配置"**(用户级偏好:每步问 vs 关键节点问 vs 不问)
- **方案 C**:保留 v2.0 二模式,**plan/ask/auto 改为"干活强度"参数**

我倾向方案 B(改动最小,符合 v2 ADR-010 HITL gate 精神:HITL 频率可调)。

---

### ⚠️ 差异点 ③ 总裁助理 vs 特别助理

用户在原文中**两个名字混用**:

- "主编排 Agent(总裁助理)"
- "对特别助理说"

**v2.0 文档统一用 "总裁助理"**。

**建议**:统一为 **总裁助理**(已在 v2.0 锁定)。
