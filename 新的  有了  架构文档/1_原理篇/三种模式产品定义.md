# 三种模式产品定义

**版本**:v1.0(对应 ADR-014;取代旧 ADR-008 的"两种模式")
**日期**:2026-05-05
**面向**:产品 / 设计 / 用户研究
**地位**:讲产品哲学、用户感知、视觉规范。**不写代码**;实现细节请看 [三种模式技术实现](../2_工程实现/三种模式技术实现.md)。

---

## 一、为什么三种模式

「有了」面对用户的不同状态:

| 用户状态 | 心理活动 | 适合的模式 |
|---------|---------|-----------|
| **想清楚要做什么但还没明确细节** | "我要做反诈视频但场景没定" | Plan(讨论)|
| **只想问问题、查信息** | "最近的诈骗趋势是什么?" | Ask(询问)|
| **目标明确,直接开干** | "做反诈视频,2026 投资理财案例" | Auto(自动)|

**关键设计**:

> **三种模式是同一个群内的工作方式,可以中途切换,不是建不同的群**(ADR-014)。

---

## 二、产品定义

### 2.1 第一种:讨论模式(Plan Mode)

**一句话**:跟 AI 团队聊清楚需求,不真做。

- 用户跟 AI 团队聊需求、想方向
- 总裁助理引导思考,**不调 Skill 工作流**
- 可以让研究员查参考资料、设计师生成快速参考图
- 后台维护 brief(需求摘要),完成度达 0.8 时主动建议切到 Auto 模式
- **不消耗任务配额**(只算 token)
- 适合:还在思考、不确定具体方向时

### 2.2 第二种:询问模式(Ask Mode)

**一句话**:问问题,AI 只答,不创建任务。

- 用户问问题,AI 团队回答,**不创建任务、不持久化结果**
- 总裁助理调用合适的 Agent 答疑
- 单轮或短轮次对话,**不走 Skill 工作流**
- **不消耗任务配额**(只算 token)
- 适合:想了解、问答、查信息时

### 2.3 第三种:自动模式(Auto Mode)

**一句话**:跟 AI 团队执行完整工作流,真做。

- AI 团队**自主执行 Skill 工作流**
- 多 Agent 协同,**走 HITL gate**(关键节点用户决策)
- 产出:完整成品(视频 / 长图 / 文档)
- **消耗配额**
- 适合:目标明确、就要做这个时

### 2.4 设计原则

**同一个群内,三种模式可中途切换**——这是核心设计。

- 用户不需要管理"应该建讨论群还是工作群"
- 群是对话单位,**模式是工作方式**
- 切换无缝:Plan 聊清楚 → Auto 直接开干,brief 自动延续

## 三、用户感知模型

```
用户进入群(主会话或工作群)
    ↓
总裁助理:"以哪种模式开始?"
[💭 讨论 Plan]  [❓ 询问 Ask]  [🚀 自动 Auto]
    ↓
用户选择 → 群顶部显示当前模式 chip → 进入工作

(随时点 chip 切换模式,或对总裁助理说"开干"/"等等再想想")
```

---

## 四、模式切换的触发方式

### 4.1 显式切换(用户主动)

- 点击群顶部"模式切换 chip" → 选 Plan / Ask / Auto
- 对总裁助理说"切到 Auto 模式" / "进入讨论模式"

### 4.2 隐式触发(总裁助理识别意图)

| 用户说 | 主编排自动切到 |
|-------|--------------|
| "开干" / "就这么做吧" / "差不多了开始" | Auto |
| "等等,我再想想" / "先不做" / "讨论一下" | Plan |
| "顺便问下" / "我有个问题" | 临时 Ask(不打断主任务)|

### 4.3 临时 Ask 模式(不打断主任务)

任何模式下用户随时可以问问题。例:

```
[Auto 模式,任务执行中]
用户:顺便问下,这种短视频抖音上一般多长?

→ 总裁助理 Ask 模式答:抖音知识科普类一般 30-60 秒效果最好。
  (不影响进行中的任务)
```

---

## 五、群形态

不再区分"讨论群" vs "工作群"。所有群:

- **通用群**:装备全部 4 个分任务 Agent,处理日常各种需求
- **专门群**:装备特定 Skill 包(反诈视频群、电商作图群)
- 主会话("你的专属 AI 工作团队"):额外含 HR + 财务经理(不进入其他群)

每个群里**三种模式都可用**,用户切换。

---

## 六、Brief 跨模式延续

**Brief**(需求摘要)在 Plan 模式产出,Auto 模式消费。

```json
{
  "完成度": 0.85,
  "字段": {
    "产品": "有机草莓",
    "受众": "城市妈妈",
    "风格": "自然质朴风,绿色调"
  },
  "决策日志": [...]
}
```

- Plan 模式聊天时,后台**防抖批量更新**(5 秒静默 / 累计 3 条)
- 完成度 ≥ 0.8 → 总裁助理主动建议:"已经聊清楚了,要不要开干?"
- 用户切到 Auto 模式 → brief 自动填充 Skill inputs,**跳过已知字段的澄清**

## 七、视觉规范

### 7.1 群顶部模式切换 chip

```
[💭 Plan]  [❓ Ask]  [🚀 Auto]
   ●          ○          ○
```

当前模式高亮(微信绿 #07C160 圆点),其他模式灰色。

### 7.2 群头像

- 主会话:浅灰底 + 团队图标
- 通用群:浅绿底 + 4 色块 Agent 头像组合
- 专门群:Skill 主题色 + Skill 图标

### 7.3 模式切换的视觉反馈

切换时群顶部出一条系统消息(类似微信"群主修改群名"):

```
─── 总裁助理 切到 自动模式(Auto) ───
```

## 八、典型场景

### 8.1 场景 A:模糊型用户(Plan → Auto)

```
用户在主会话:"我想做电商详情图"
    ↓
总裁助理:[💭 Plan] [❓ Ask] [🚀 Auto]
    ↓
用户点 Plan → 进入"专属 AI 工作团队"群,模式 = Plan
    ↓
"你想做什么样的电商图?产品和受众是?"
(讨论 5 分钟,brief 完成度 → 0.85)
    ↓
总裁助理:"已经聊清楚了,要不要开干?"
    ↓
用户点 [切到 Auto] → 同群内 work_mode = auto
    ↓
"已应用讨论中聊清楚的需求(产品=有机草莓...),开始制作。"
工作流执行 → 最终交付
```

### 8.2 场景 B:明确型用户(直接 Auto)

```
用户在主会话:"做朴朴风的有机草莓电商详情图"
    ↓
总裁助理识别明确意图 → 询问"建专门群?"
[新建电商作图群,Auto 模式开始制作]
    ↓
工作流直接执行 → 最终交付
```

### 8.3 场景 C:中途切回 Plan

```
[工作群,Auto 模式执行中,T+30s]
用户:"等等,我觉得方向不对,要重新想想"
    ↓
总裁助理识别 → 暂停任务 + 切到 Plan 模式
"好,先暂停。我们继续聊吧,刚才聊到:..."
    ↓
[继续讨论]
    ↓
用户:"行,按这个新方向开始"
    ↓
切回 Auto,brief 已含新方向,继续制作
```

### 8.4 场景 D:任意时刻临时 Ask

```
[任意模式,任意时刻]
用户:"顺便问下,小红书的封面图最佳尺寸是?"
    ↓
总裁助理 Ask 模式回答:"小红书封面推荐 3:4 或 1:1..."
    ↓
回到原模式,主任务不受影响
```

## 九、模式偏好沉淀

| 用户行为 | 偏好状态 |
|---------|---------|
| 第 1 次新建群,选 X 模式 | 不影响下次 |
| 连续 2 次同选 X 模式 | 下次默认显示 X(仍要点确认)|
| 连续 3 次同选 X 模式 | 下次直接进 X 模式(跳过选择)|
| 之后选了不同模式 | 偏好重置(回到第 1 次状态)|

存储在 `user_preferences.default_work_mode`。

---

## 十、计费与限额

| 项 | Plan 模式 | Ask 模式 | Auto 模式 |
|----|----------|---------|----------|
| 任务配额 | 不消耗 | 不消耗 | 消耗 |
| Token 计费 | 是(便宜)| 是(便宜)| 是(标准)|
| 配额扣费时机 | - | - | Skill 启动时 |
| 视频任务额外限额 | - | - | 每日 N 次 |

## 十一、V1 / V2 边界

### V1 必做

- 三种模式同群内切换(ADR-014)
- 模式切换 chip + 显式 / 隐式触发
- Brief 在 Plan 模式后台维护(防抖批量,ADR-006)
- Brief 跨模式延续(填 Auto 字段)
- 偏好沉淀(`default_work_mode`)
- 临时 Ask(不打断主任务)

### V2 推迟

- 多人协作群下的模式同步
- Plan 模式跨群参考(把别群的 brief 拉过来)
