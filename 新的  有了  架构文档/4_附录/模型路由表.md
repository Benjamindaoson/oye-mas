# 模型路由表(L1 + L2)

**版本**:v3.0(对应 ADR-007 + ADR-001-rev;hero SKU = 反诈视频 + 电商详情图)
**日期**:2026-05-05
**面向**:工程团队 / SRE / 成本审计
**地位**:LiteLLM 配置的唯一来源。任何模型选型变更必须先改本文档。

**v3.0 关键变更**:
- 🔄 Agent 章节按 ADR-001-rev 编号反向(Agent 2=文档 / Agent 3=图 / Agent 4=影音)
- 🔄 短视频 task_type 保留(V1.5 通过 Skill 市场使用)
- 🔄 反诈视频 task_type 沿用 long_writing / web_search / video_compose 等通用 task_type
- 保留:Reflexion / Skill Drafter 走 L1 / HITL 解析 / Brief 更新

---

## 一、设计原则(ADR-007)

**两层路由,各管各的**:

| 层 | 用途 | 共同要求 |
|----|------|---------|
| **L1 编排层** | 主编排子模块的 LLM 调用 | 快、便宜、JSON 输出、低延迟 |
| **L2 执行层** | 4 个分任务 Agent 的 LLM 调用 | 按 task_type 选最强模型,接受较高延迟 |

两层的健康度 / 降级策略**独立**,互不干扰。

---

## 二、L1 编排层路由

主编排子模块的所有 LLM 调用走 L1。

### 2.1 调用点

| 调用点 | task_type 标识 | 上下文大小 | 期望延迟 |
|-------|---------------|-----------|---------|
| 意图理解器 | `intent_understanding` | 1500 tokens | 300ms |
| Skill 匹配器(LLM 兜底)| `skill_matching` | 1500 tokens | 600-800ms |
| 中断处理器(分类)| `interrupt_classification` | 1500 tokens | 300ms |
| 模式管理器(brief 更新)| `brief_update` | 1000 tokens | 300ms |
| **HITL 用户决定解析**(v2.0)| `hitl_decision_parsing` | 800 tokens | 200ms |
| **回滚目标计算**(v2.0,中断 C)| `rollback_target_compute` | 1000 tokens | 200ms |
| **MCP tool 调用决策**(v2.0)| `mcp_tool_decision` | 2000 tokens | 500ms |
| 澄清版本生成(转 L2)| `version_compare` | 1000 tokens | 3-5s |

### 2.2 模型选型

| 主力 | 备用 1 | 备用 2 |
|------|--------|--------|
| **deepseek-v4-flash** | claude-haiku-4-5 | gpt-5-mini |

**理由**:

- DeepSeek-v4-flash 中文场景成本/性能最优,JSON 输出稳定
- Haiku 4.5 在 DeepSeek 宕机时能保证服务可用
- gpt-5-mini 作为最后兜底(双备援)

### 2.3 LiteLLM 配置示例

```yaml
# litellm_config_l1.yaml
model_list:
  - model_name: l1_orchestration_primary
    litellm_params:
      model: openai/deepseek-v4-flash
      api_base: https://api.deepseek.com/v1
      api_key: os.environ/DEEPSEEK_API_KEY
      max_tokens: 500
      temperature: 0.3
      response_format: {"type": "json_object"}
      timeout: 2

  - model_name: l1_orchestration_fallback_1
    litellm_params:
      model: anthropic/claude-haiku-4-5
      api_key: os.environ/ANTHROPIC_API_KEY
      max_tokens: 500
      temperature: 0.3

  - model_name: l1_orchestration_fallback_2
    litellm_params:
      model: openai/gpt-5-mini
      api_key: os.environ/OPENAI_API_KEY
      max_tokens: 500
      temperature: 0.3

router_settings:
  routing_strategy: simple-shuffle      # 单实例
  fallback_chains:
    - [l1_orchestration_primary, l1_orchestration_fallback_1, l1_orchestration_fallback_2]
  num_retries: 1
  timeout: 2
  cache_responses: false                # 编排层不缓存(每次都不同)
```

### 2.4 健康度切换

- **主力错误率 > 5%(过去 5 分钟)** → 切备用 1
- **备用 1 错误率 > 5%** → 切备用 2
- **每 60 秒重新探测主力** → 健康则切回

### 2.5 成本预估

| 调用点 | 频次/任务 | 单次成本 | 任务总成本 |
|-------|----------|---------|----------|
| 意图理解 | 1-2 次 | $0.0002 | $0.0002-0.0004 |
| Skill 匹配 LLM 兜底 | 0-1 次 | $0.0002 | $0-0.0002 |
| 中断分类 | 0-3 次 | $0.0002 | $0-0.0006 |
| Brief 更新(讨论群)| 3-5 次/讨论 | $0.0002 | $0.0006-0.001 |

**主编排单任务总成本上限**:$0.003

## 八、统一的 routing_hints 字段

任何 step 都可在 `routing_hints` 里覆盖默认选型:

```yaml
routing_hints:
  primary: kimi-k2                       # 主力(覆盖默认)
  fallback: [deepseek-v4-pro, sonnet]    # 备用链
  language: zh                           # 语言(影响默认选型)
  cost_priority: low | medium | high     # 成本敏感度
  style_category: realistic | cartoon | chinese  # (Agent 2 专用)
  voice: female_warm | male_serious      # (Agent 3 TTS 专用)
```
