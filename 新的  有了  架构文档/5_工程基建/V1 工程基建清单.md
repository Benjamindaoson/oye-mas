## 1. 认证 / 授权

### 1.1 选型

- **JWT** Access Token(15 分钟) + Refresh Token(30 天)
- **手机号 + 短信验证码**为主流注册方式
- 工具:**FastAPI Users 12.x** 集成 SQLAlchemy + JWT

### 1.2 流程

```
[注册 / 登录]
用户输入手机号 → 服务端发短信(阿里云短信)→ 用户输入验证码
→ 服务端校验 → 生成 access_token + refresh_token
→ 返回客户端 → 存 localStorage(refresh)+ 内存(access)

[API 鉴权]
客户端每次请求带 Authorization: Bearer <access_token>
中间件校验 → 注入 current_user 到 request.state

[Token 刷新]
access_token 过期 → 客户端用 refresh_token 调 /api/auth/refresh
→ 新 access_token + 新 refresh_token(rotation)

[登出]
清除客户端 token + 服务端把 refresh_token 加入 Redis 黑名单(直到自然过期)

[WebSocket 鉴权]
WS 连接时 query 参数带 token,握手阶段校验
```

### 1.3 中间件实现

```python
# app/middleware/auth.py
from fastapi import Depends, HTTPException
from fastapi_users.authentication import JWTStrategy
from app.models.user import User

async def get_current_user(
    request: Request,
    jwt_strategy: JWTStrategy = Depends(),
) -> User:
    auth_header = request.headers.get("Authorization", "")
    if not auth_header.startswith("Bearer "):
        raise HTTPException(401, "Missing token")
    
    token = auth_header[7:]
    user = await jwt_strategy.read_token(token, user_manager)
    
    if not user or not user.is_active:
        raise HTTPException(401, "Invalid or inactive user")
    
    request.state.user = user
    return user

# 用法
@router.get("/api/conversations")
async def list_conversations(user: User = Depends(get_current_user)):
    return await ConversationService.list_for_user(user.id)
```

### 1.4 API 端点

```
POST /api/auth/sms/send       发短信验证码
POST /api/auth/register       注册(手机号 + 验证码)
POST /api/auth/login          登录(手机号 + 验证码)
POST /api/auth/refresh        刷新 access_token
POST /api/auth/logout         登出
GET  /api/auth/me             获取当前用户

POST /api/auth/oauth/wechat   微信 OAuth(V1.5)
```

### 1.5 安全要点

- 短信验证码:6 位数字,5 分钟有效,Redis 存,同手机号 1 分钟内只能发 1 次
- access_token 在 Redis 不存(stateless),refresh_token 存 Redis(便于撤销)
- 密码字段不存(本产品不用密码登录)
- 敏感操作(改密码、删账号)需 step-up:再发一次短信验证码

## 2. WebSocket 网关

### 2.1 选型

- **FastAPI 原生 WebSocket** + **Redis Pub/Sub** 多实例 fanout
- 不引入 Socket.io(过重)

### 2.2 架构

```
浏览器                     后端 1                后端 2
   │                         │                     │
   ├──WS handshake──────────>│                     │
   │  (含 access_token)      │                     │
   │                         │                     │
   │<─────WS message────────│                     │
   │   (实时事件)            │                     │
   │                         │                     │
                             ↑                     ↑
                             │                     │
                             └─── Redis Pub/Sub ───┘
                                  channel: user:{user_id}

后端任意实例发布消息到 Redis channel
→ 持有该用户连接的实例订阅到 → 通过 WS 推给浏览器
```

### 2.3 实现

```python
# app/ws/connection.py
import asyncio
import json
from fastapi import WebSocket, WebSocketDisconnect, Depends
from redis.asyncio import Redis

class ConnectionManager:
    def __init__(self):
        self.local_connections: dict[str, WebSocket] = {}
    
    async def connect(self, ws: WebSocket, user_id: str, redis: Redis):
        await ws.accept()
        self.local_connections[user_id] = ws
        
        # 订阅 Redis pubsub
        pubsub = redis.pubsub()
        await pubsub.subscribe(f"user:{user_id}")
        
        # 启动消费协程
        consumer = asyncio.create_task(self._consume(ws, pubsub))
        
        try:
            while True:
                # 客户端发来的消息(暂时只处理 ping)
                data = await ws.receive_text()
                if data == "ping":
                    await ws.send_text("pong")
        except WebSocketDisconnect:
            consumer.cancel()
            await pubsub.unsubscribe(f"user:{user_id}")
            del self.local_connections[user_id]
    
    async def _consume(self, ws: WebSocket, pubsub):
        async for msg in pubsub.listen():
            if msg["type"] == "message":
                await ws.send_text(msg["data"])

manager = ConnectionManager()


# WS 端点
@app.websocket("/ws")
async def websocket_endpoint(
    ws: WebSocket,
    token: str,                          # query param
    redis: Redis = Depends(get_redis),
):
    user = await verify_token(token)
    if not user:
        await ws.close(code=4001)
        return
    await manager.connect(ws, user.id, redis)


# 任意位置发送事件
async def emit_to_user(user_id: str, event: dict, redis: Redis):
    await redis.publish(f"user:{user_id}", json.dumps(event))
```

### 2.5 重连与缓冲

- 客户端断线后 5 秒自动重连
- 服务端对每个 user 维护 ringbuffer(最近 100 条事件,Redis Stream)
- 重连时客户端带 `last_event_id` query 参数,服务端把缺失事件补发

```
GET /ws?token=xxx&last_event_id=1715000000-0
```

# app/services/oss.py
import oss2
from datetime import timedelta

class OSSService:
    def __init__(self, settings):
        auth = oss2.Auth(settings.OSS_ACCESS_KEY, settings.OSS_SECRET_KEY)
        self.bucket = oss2.Bucket(auth, settings.OSS_ENDPOINT, settings.OSS_BUCKET)
    
    async def generate_put_url(
        self, 
        path: str,
        max_size_mb: int = 100,
        expires: timedelta = timedelta(minutes=15)
    ) -> str:
        return self.bucket.sign_url(
            "PUT",
            path,
            expires.total_seconds(),
            headers={"Content-Length": str(max_size_mb * 1024 * 1024)},
        )
    
    async def generate_get_url(
        self,
        path: str,
        expires: timedelta = timedelta(hours=1),
    ) -> str:
        return self.bucket.sign_url("GET", path, expires.total_seconds())
    
    async def upload_bytes(self, path: str, data: bytes) -> str:
        self.bucket.put_object(path, data)
        return f"oss://{self.bucket.bucket_name}/{path}"
    
    async def download_bytes(self, oss_url: str) -> bytes:
        # oss://bucket/path → path
        path = oss_url.split("/", 3)[-1]
        return self.bucket.get_object(path).read()
```

# app/services/quota.py
from datetime import date

class QuotaService:
    async def check_and_consume(
        self, user_id: str, quota_type: str, amount: int = 1
    ) -> tuple[bool, int, int]:
        """
        Returns: (success, current_used, limit)
        """
        period = self._get_period(quota_type)            # 日 / 月
        limit = self._get_limit(user_id, quota_type)     # 按用户 plan
        
        key = f"quota:{user_id}:{quota_type}:{period}"
        
        # Redis Lua 原子操作:check + incr
        lua = """
        local current = tonumber(redis.call('GET', KEYS[1]) or '0')
        local limit = tonumber(ARGV[1])
        local amount = tonumber(ARGV[2])
        if current + amount > limit then
            return {0, current, limit}
        end
        local new_val = redis.call('INCRBY', KEYS[1], amount)
        redis.call('EXPIRE', KEYS[1], ARGV[3])
        return {1, new_val, limit}
        """
        ttl_seconds = self._get_ttl(quota_type)
        result = await redis.eval(lua, 1, key, limit, amount, ttl_seconds)
        success = bool(result[0])
        
        # 异步同步到 PostgreSQL 用于审计
        if success:
            await self._async_persist(user_id, quota_type, period, amount)
        
        return success, result[1], result[2]
    
    async def warn_if_low(self, user_id: str, quota_type: str):
        """剩余 < 20% 时通过 WS 推送警告"""

alembic upgrade head

# 回滚
alembic downgrade -1
```

### 14.2 上线时机

- 数据库迁移**先于**应用部署执行
- 向后兼容(老应用能跑新 schema 至少 1 个版本)
- 大表 DDL 用 `pg_repack` 避免锁

## 17. 错误码体系

### 17.1 统一格式

```json
{
  "error": {
    "code": "QUOTA_EXCEEDED_VIDEO_DAILY",
    "message": "你今天的视频任务配额已用完",
    "user_facing": true,
    "details": {
      "quota_type": "video_task_daily",
      "used": 3,
      "limit": 3,
      "reset_at": "2026-05-06T00:00:00+08:00"
    },
    "retry_after_seconds": 3600
  }
}
```

### 17.2 错误码命名

`{大类}_{子类}_{具体}`:

| 前缀 | 大类 |
|------|------|
| `AUTH_` | 认证 |
| `QUOTA_` | 配额 |
| `VALIDATION_` | 输入校验 |
| `SKILL_` | Skill 相关 |
| `AGENT_` | Agent 错误 |
| `MODEL_` | 模型调用 |
| `OSS_` | 对象存储 |
| `INTERNAL_` | 内部错误 |

例:

- `AUTH_TOKEN_EXPIRED`
- `QUOTA_EXCEEDED_VIDEO_DAILY`
- `VALIDATION_FIELD_MISSING`
- `SKILL_NOT_FOUND`
- `AGENT_TASK_TIMEOUT`
- `MODEL_RATE_LIMITED`
- `OSS_UPLOAD_TOO_LARGE`
- `INTERNAL_DATABASE_ERROR`

### 17.3 用户面文案映射

每个 `user_facing: true` 的错误码必须在 `4_附录/系统 Prompt 全集.md §7` 有对应文案。

## 7.7 Reflexion + Skill Drafter Pipeline(v2.0,ADR-011)

### 7.7.1 Reflexion(失败 → prompt 改进)

```python
# Celery 任务
@celery_app.task
def run_reflexion(task_id):
    task = load_task(task_id)
    failure = extract_failure(task)
    
    analysis = call_litellm(
        model="claude-sonnet-4-6",
        messages=[
            {"role": "system", "content": REFLEXION_SYSTEM_PROMPT},
            {"role": "user", "content": build_context(task, failure)}
        ],
        response_format={"type": "json_object"}
    )
    
    save_to_prompt_improvement_candidates(analysis)
    notify_team_via_slack("New prompt improvement candidate")
```

触发:任务失败 / 用户负面反馈 / HITL 拒绝时,主编排发 Celery 任务到 `flywheel_tasks` 队列。

### 7.7.2 Skill Drafter(高满意度 → Skill 草稿)

```python
@celery_app.task
def run_skill_drafter(task_id):
    task = load_task(task_id)
    if task.user_satisfaction < 4:
        return
    
    if not is_novel_workflow(task):
        return
    
    draft_yaml = call_litellm(
        model="claude-sonnet-4-6",
        messages=[
            {"role": "system", "content": SKILL_DRAFTER_PROMPT},
            {"role": "user", "content": build_trace_context(task)}
        ]
    )
    
    if validate_skill_yaml(draft_yaml):
        save_to_skill_drafts(task.user_id, task.id, draft_yaml)
        notify_user_via_ws(task.user_id, "你刚才那个工作流可以发布为 Skill,要试试吗?")
```

### 7.7.3 配额

每个用户每天最多 3 次 Reflexion / 1 次 Skill Drafter(避免连环失败时刷爆成本)。
