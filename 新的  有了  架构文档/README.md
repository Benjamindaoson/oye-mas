### 三步起项目

```bash
# 1. 复制 3 份到目标仓库
mkdir -p /path/to/youle-repo/docs
cp "新的  有了  架构文档/Claude Code 启动总提示词.md" /path/to/youle-repo/CLAUDE.md
cp "新的  有了  架构文档/启动配套/ARCHITECTURE.md" /path/to/youle-repo/docs/ARCHITECTURE.md
cp "新的  有了  架构文档/启动配套/CONSTITUTION.md" /path/to/youle-repo/docs/CONSTITUTION.md

# 2. 在仓库下启动 Claude Code
cd /path/to/youle-repo && claude

# 3. 第一句话告诉 Claude:"按 CLAUDE.md 的 Sprint 0 起骨架"
```

Claude Code 默认读 CLAUDE.md;遇到具体技术问题它会查 ARCHITECTURE.md;有人想偏离原则时它去查 CONSTITUTION.md。三层各司其职,context 不浪费。

## 维护规则

1. **本目录是唯一真理来源**——任何团队成员引用文档时,从本 README 入口跳转
2. **变更治理**:
   - 产品定义变更 → 更新 `0_总览/` + `1_原理篇/`
   - 工程实现变更 → 更新 `2_工程实现/` + `4_附录/` + `5_工程基建/`
   - 涉及架构判断 → 必须新增 ADR(`3_决策记录/`)
   - 商业 / 单位经济变更 → 更新 `3_决策记录/商业模型与单位经济.md`
3. **单一事实源**:不允许同一信息在多份文档中重复维护;交叉引用即可
4. **每份文档顶部都标明:版本 / 日期 / 面向 / 地位**
5. **任何 PR 必须验证 4 类飞轮信号沉淀 + HITL gate 合规**(CI 强制)
