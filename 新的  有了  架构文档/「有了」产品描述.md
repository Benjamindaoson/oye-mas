# 「有了」产品描述

「有了」是你的专属 AI 工作团队。第一期重点针对"内容引流"和"营销"场景。整个产品的外观和交互 80% 像微信。

## 一、产品定位

核心是**群内的多智能体协作**——产品的本质是**一支 AI 团队为你交付专业结果**。

### 用户为什么用

一句话:用更低的成本、更短的时间,拿到一次到位的专业结果。

- 操作便捷:自然语言下达需求,不用学 Prompt 工程
- 响应迅速:多 Agent 并行,不卡顿、不空等
- 一次到位:成果直接可用,不改或仅小改
- 降本增效:替代专业团队(设计师/文案/剪辑)

### 与 GPT / Claude / DeepSeek 的差异

通用大模型的两个问题:多次交互才能拿到结果且常不满意,用户反复修改的时间≈自己做;直接调大模型成本高。

本产品由总裁助理一次理解意图、调 Skill 工作流、多 Agent 协同交付完整成品。**用户从"和 AI 反复来回"变成"对 AI 团队下达任务"**——专业结果一次到位。

## 二、系统架构

系统的核心竞争力是**意图理解、意图澄清、任务编排**——把用户模糊的需求精准转化为结构化任务,并交付符合甚至超出预期的结果。

系统由两部分组成:

**不变的部分**:多智能体框架(总裁助理 + 4 个分任务 Agent + 2 个支持 Agent)

**变的部分**:Prompt 和 Skill

- Prompt 是单步骤提示词(小颗粒),用户可收藏到知识库
- Skill 是完整工作流(大颗粒),由多步骤、多 Agent 协作组成

### Skill 库

- 来源:平台预置 + 创作者贡献
- 用户不直接配置 Skill
- 总裁助理在理解意图后自动从 Skill 库匹配并调用,分配给对应 Agent 执行

### 创作者飞轮

用户可以把自己的 Prompt、群内工作流或对话历史总结为新的 Skill。只需对总裁助理说"我要成为 Skill 创作者",总裁助理引导:自动分析用户最近工作流 → 自动总结成 Skill 草稿 → 用户审核确认 → 发布。

但**审核机制不能省**:发布前平台必须审核(质量 + 不含违规 prompt),避免市场被低质 Skill 污染。

## 三、Agent 团队

### 3.1 主编排 Agent

**总裁助理**:用户的贴身助手,永远在场,不直接产出内容,负责意图理解、Skill 匹配、任务分配、中断处理。每个用户只有一个总裁助理,跟着用户走——主会话和所有群里都是同一个。

### 3.2 4 个分任务 Agent(干活的)

群内每个 Agent 是一个数字员工。不同的群岗位名字可能不一样,但底层技术 Agent 通用。

**Agent 1 = 研究员 / 文案师**(文字 + 工具调用)

同一个 Agent 两个角色显示——做调研叫"研究员",做写作叫"文案师"。

核心能力:

- 文字生成:短文、长文、结构化(大纲、提案)、摘要、改写、润色、翻译
- 研究与信息收集:联网搜索(Tavily)、网页抓取(Playwright)、数据收集与整合、信息源溯源
- 阅读理解与分析:文档理解、信息提取、推理、多文档对比

边界:不画图、不做视频音频、不组装办公文档

**Agent 2 = 文档专员**(办公文档)

核心能力:

- PPT:生成、修改、提取、模板应用
- Excel:读写、公式、图表、格式化
- Word:生成、修改、提取、格式化
- PDF:文字 / 表格提取、生成、加水印、扫描 OCR
- 长图垂直拼接(电商详情图最后一步)

边界:不写文字内容、不生成图、不做视频

**Agent 3 = 设计师**(图像)

核心能力:

- 图像生成:文生图、图生图、批量生成(保持风格一致)、多图合成
- 图像理解:图片描述、风格分析、质量评估、OCR
- 图像编辑:局部重绘、扩图、抠图、画质增强、图片下载
- 风格管理:从参考图提取、应用预设风格库、风格迁移

核心能力点(电商详情图必备):**批量生成时第 1 张为视觉锚点,后续保持风格统一**

边界:不写文字、不做视频、不组装文档

**Agent 4 = 影音师**(视频与音频)

核心能力:

- 视频生成:文生视频、图生视频、视频合成
- 视频理解:描述、关键帧提取
- 视频编辑:剪辑、字幕、BGM、转场
- 音频处理:TTS 配音、Whisper 识别、BGM 选择、字幕对齐

核心能力点(反诈视频必备):**视频合成是 5-10 分钟的长任务,异步执行不阻塞**

边界:不写脚本、不画静态图、不做办公文档

### 3.3 2 个支持 Agent(管理类)

这两个 Agent 一直在主会话里,用户随时可以叫:

**HR**:管理用户的 AI 团队

- 用户问"我应该用什么 Agent 处理这件事" → HR 推荐
- 用户想给 Agent 进修(喂专属语料让 Agent 升级,V2)→ HR 引导
- 用户想加新 Skill → HR 协助挑选
- 用户想成为 Skill 创作者 → HR 引导(V2)

**财务经理**:管订阅、配额、成本

- 用户问"还剩多少配额" → 财务经理回答
- 预算紧张时主动提醒
- 用户想升级 / 续费 → 财务经理处理
- 月度账单总结 → 财务经理汇报

### 3.4 Agent 协作规则

- Agent 之间通过总裁助理协调,不直接派活
- 用户视角能看到 Agent 之间的互动对话(由总裁助理编排生成),感觉像真实工作群
- Agent 间数据通过产物引用传递,不直接对话
- 每个 Agent 有边界,超出能力必须返错(例:Agent 1 收到画图任务 → 返错"我是文字 Agent,请让设计师处理")

## 四、Agent 拟人化(让 AI 员工有"公司感")

### 4.1 设计哲学

每个 Agent 不仅是工具,更是用户可了解、信任、深入互动的数字员工。通过履历卡片、悬停交互、私聊、表情系统、状态显示实现 AI 角色的人格化和 IP 化。

### 4.2 表情系统(V1 做)

每个 Agent 有自己的 20 组表情包(喜悦、思考、沮丧等),**只在关键节点出现**:

- 触发节点:成果产出、事情办不到、用户表扬、用户批评
- **严肃场景(金融、医疗等)不带表情**
- 内置情绪系统,有必要时才出现表情(避免做作)

### 4.3 工作状态显示(V1 做)

每个 Agent 头像下方显示当前状态:

- "工作中..."(执行任务)
- "发呆中..."(空闲)
- "摸鱼中..."(空闲超过 30 分钟)
- "进修中..."(用户给 Agent 喂语料,V2)

让 Agent 更有人格,不是冰冷的工具。

### 4.4 Agent 之间的互动

中间对话栏的 Agent 之间互动要多一些,接近真实公司工作群的样子。

例:

```
[研究员] @文案师 调研报告做完了,8 个案例都整理好了,
        参考来源都附在表里。看你的脚本了。

[文案师] 收到 @研究员,马上开始写。
        @设计师 等会儿脚本里会有 5 张图的描述,
        到时候麻烦你处理。

[设计师] 好,我先准备风格库 [思考表情]
```

实现层面:这些"互动"由总裁助理在派活时统一编排——看起来像 Agent 间对话,实际是总裁助理在调度。这样既有"真公司"的体验,工程层又保持架构原则不变。

### 4.5 悬停交互:Agent 履历卡片

触发:鼠标悬停 Agent 头像 0.3 秒后浮现。

卡片内容:

- 基础信息:头像 + 名称 + 一句话能力定位 + 背后模型
- 专业标识:3-5 个领域标签 + 进修记录(如有)
- 服务统计:个人("已为你完成 47 次") + 平台("累计 120 万次")
- 操作按钮:[私聊] [查看更多]

### 4.6 私聊

触发(两种等效):双击 Agent 头像 / 点履历卡片 [私聊] 按钮。

特征:

- 不启动完整 Skill 工作流
- 不调度其他 Agent
- 响应更快(跳过编排环节)
- 对话历史独立(与群聊互不污染)
- 仍可调用单 Agent 的轻量 Skill

群聊 vs 私聊:

- 群聊:多步骤、跨能力的完整任务,多 Agent 协同
- 私聊:单一能力、快速响应,单 Agent 直接执行

### 4.7 Agent 名片分享

用户可将 Agent 履历卡片作为"名片"分享:朋友点击 → 看履历 → 一键进入私聊 → 引导注册。**意义**:零成本获客飞轮,AI 员工社交化。

## 五、产品的核心交互

### 5.1 三栏布局

- 左栏:会话列表 + 应用导航
- 中栏:对话主战场(群聊框)——产品的主时间线,所有历史都在这
- 右栏:执行流(任务执行时的临时展示,核心差异化)

### 5.2 右栏执行流(核心差异化)

让用户实时看到 AI 团队怎么协作:

- 三层抽屉式进度(阶段总览 / 子任务结构 / 实时细节)
- 状态符号:✓ 完成 / ⟳ 进行中 / ○ 未开始
- 任务进行中自动展开,任务结束自动收起

### 5.3 群的形态

- 通用群:装备全部 4 个分任务 Agent
- 专门群:装备特定 Skill 包(反诈视频群、电商作图群)
- 每个群有总裁助理在场
- 群头像默认"我的世界"像素风或 NFT 风格

### 5.4 @ 调用机制

输入"@"弹出列表,3 类:

- @ 人(AI 员工)
- @ 文件(素材库)
- @ 提示词(知识库)

### 5.5 视觉风格

三栏极简布局;微信绿 #07c160 唯一强调色;大量留白;线条 SVG 图标,无 emoji;头像统一"我的世界"像素风 / NFT 风格。

## 六、三种工作模式

总裁助理在用户开始任务时,确认以哪种模式工作。**三种模式是同一个群内的不同工作方式,可以中途切换,不是建不同的群**。

**第一种:讨论模式(Plan Mode)**

- 用户跟 AI 团队聊需求、想方向
- 总裁助理引导思考,不动手做
- 适合:还在思考、不确定具体方向时

**第二种:询问模式(Ask Mode)**

- 用户问问题,AI 只回答,不创建任务
- 总裁助理调用 Agent 查信息、答疑
- 适合:想了解、问答、查信息时

**第三种:自动模式(Auto Mode)**

- AI 团队自主执行 Skill 工作流
- 多 Agent 协同,用户只在关键决策点澄清
- 适合:目标明确、就要做这个时

模式切换:

- 用户在 Plan 模式聊清楚后,可以说"开干"切换到 Auto 模式
- Auto 模式中用户说"等等,我再想想"可以回到 Plan 模式
- 任何模式下用户可以随时问问题(临时 Ask 模式),不影响主任务

## 七、首次进入的体验

用户第一次进入「有了」,自动进入一个群,叫做"**你的专属 AI 工作团队**"。

像加入微信群一样,依次显示:

```
[14:00] 总裁助理 加入群聊
[14:00] HR 加入群聊
[14:00] 财务经理 加入群聊

[14:01] 总裁助理:你好,我是你的总裁助理,
        平时帮你协调任务、调度团队。

[14:01] HR:我管理你的 AI 团队成员,
        以后想加新员工或给员工进修,找我。

[14:01] 财务经理:订阅、配额、账单都归我。
        预算紧张时我会提醒你。

[14:02] 总裁助理:你想以哪种模式开始工作?
        [讨论模式] [询问模式] [自动模式]
        [简单了解每种模式]
```

用户选择模式后,正式开始工作。

这个"专属 AI 工作团队"群是用户的主会话:

- 后续元操作(创建新群、订阅 Skill、跨群协调)都在这里和总裁助理对话完成
- HR 和财务经理一直在,用户随时可以 @
- 不做传统 onboarding tour——用户直接对话上手

## 八、左栏导航(简化版)

从上到下:

**顶部 - 用户身份与数据**

1. **用户头像**——默认像素风/NFT 风格,可上传;点击进入个人主页(信息+订阅+偏好)
2. **知识库**——用户收藏的 Prompt;在聊天中通过 @ 提示词名调用
3. **成果库**——所有群产出的成果统一保存,跨群可访问,支持下载、分享、二次编辑

**中部 - 主功能**

1. **聊天**——会话列表(默认入口);"专属 AI 工作团队"主会话置顶;红点显示未读
2. **AI 学院**——浏览 Skill、Skill 管理、Skill 创作(V2)、Agent 进修(V2)
3. **技能市场**——浏览订阅 Skill、创作者计划、创作者后台

**底部**

1. **设置**——通知、隐私、语言、主题、帮助

不要"通讯录"等多余入口。

## 九、卡片式分享(获客飞轮)

设计原则:产品核心是群聊,不是单个 Agent。**优先级:群 > 成果 > 单 Agent**。

### 9.1 群名片(核心)

卡片内容:

- 基础信息:群头像 + 群名 + 群简介
- 团队组成:Agent 列表 + 每个角色
- 装备 Skill:订阅或预置的 Skill 列表
- 战绩数据:累计交付数 + 群成就

操作按钮:[加入此团队] [查看成果] [关注创作者]

[加入此团队]:朋友一键复制相同配置(Agent + Skill + 偏好),零摸索成本

### 9.2 成果分享(核心)

卡片内容:

- 成果本体:缩略图 / 预览 / 可播放
- 产出溯源:哪个群、哪些 Agent、什么 Skill、耗时

操作按钮:[下载] [做同款] [查看创作者]

[做同款]:自动创建相同配置的群,引导用户填入自己的参数,5 分钟产出用户自己的版本。**从"看到成果"到"拥有同款"路径极短**。

### 9.3 单 Agent 名片(次要)

详见第 4 章。非主推。

### 9.4 增长机制

```
用户产出好成果 → 分享群/成果 → 朋友看到 
→ 一键做同款 / 加入团队 → 朋友也产出 → 再次分享 
→ 形成"群作为产品单位"的扩散
```

每次分享 = 产品价值传递 + 产品功能演示。

## 十、典型应用场景(V1 上线 2 个)

### 10.1 反诈视频制作

总裁助理调用「反诈视频制作」Skill,工作流:

**第 1 步:案例调研**(Agent 1 → 联网搜索)

Skill 内置 Prompt(含变量 {年份}、{骗局类型}):

> 给我 10 条 {年份} 涉及 {骗局类型}、传销、网络洗钱的案件新闻,并且罗列涉案金额、新闻简介 50 字。标题要有夸张感,模仿抖音标题。给我新闻来源网址。生成 excel 表格。只要有图片的来源。对于有图片的新闻来源,直接把新闻中的图片复制到 excel 表格单独一列。

输出:xlsx 表格

**第 2 步:脚本生成**(Agent 1)

基于 xlsx 生成视频脚本(Word);总裁助理发起选择题澄清(≤ 5 轮)

**第 3 步:图片处理**(Agent 1 + Agent 3)

从 xlsx 下载图片,自动命名,质量检查;输出 5 张配图

**第 4 步:配乐生成**(Agent 4)

基于脚本情绪选 BGM;输出 mp3

**第 5 步:自动剪辑**(Agent 4)

整合脚本、图、配音、BGM、字幕;输出 mp4

**最终交付**:mp4 + xlsx + docx + 5 张配图 + mp3

### 10.2 电商详情图制作

**第 1 步**:素材输入(用户上传商品图 + 卖点 + 风格选择)

**第 2 步**:风格分析(Agent 3)→ 风格指引

**第 3 步**:文案撰写(Agent 1)→ 5 段文案

**第 4 步**:分段图生成(Agent 3)→ 5 张分段图(第 1 张视觉锚点,保持风格统一)

**第 5 步**:长图拼接(Agent 2)→ 1 张长图

**第 6 步**:质量校验(Agent 3)→ 不达标自动重做

**最终交付**:1 张精美的电商详情长图

### 10.3 关键设计共性

- 用户输入是模糊的(如"做反诈视频"、"做朴朴风格")
- 总裁助理负责理解和编排(自动匹配 Skill + 分配任务)
- 多 Agent 协同(每场景 2-4 个分任务 Agent 协作)
- 关键决策点向用户澄清(永远选择题,≤ 5 轮,不让填空)
- 最终交付的是完整产物(成片或成图,拿到就能用)

### 10.4 关键技术原则

- Prompt 是 Skill 内置的,含变量插槽
- 总裁助理自动填充变量并调用
- 用户只说自然语言,不写 prompt



**架构层**:

- 总裁助理 + 4 个分任务 Agent + 2 个支持 Agent(共 7 个 AI 角色)
- Agent 2 = 文档,Agent 3 = 图,Agent 4 = 视频(已修正反诈视频和电商详情图场景的对应)

**交互层**:

- 三种工作模式(Plan/Ask/Auto)是同群内的工作方式,可中途切换
- 首次进入"专属 AI 工作团队"群,3 个角色依次入群
- HR 和财务经理常驻主会话

**人格化层**:

- 表情系统 V1 做,20 组,关键节点出现,严肃场景关闭
- 工作状态显示 V1 做("摸鱼中"等)
- Agent 之间用户视角互动多,工程层仍由总裁助理编排

**导航层**:

- 左栏简化:头像 + 知识库 + 成果库 提到顶部
- 取消"通讯录"等多余入口
